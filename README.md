# TENG-Based Acoustic Vibration Word Recognition using Deep Learning

## Overview

This project demonstrates a sensing system where a **Triboelectric Nanogenerator (TENG)** converts **mechanical vibrations generated by a speaker into electrical signals**, which are then classified using a **1D Convolutional Neural Network (CNN)**.

Instead of using microphones, this system recognizes words by detecting **vibration patterns produced by sound waves**. These vibrations excite the TENG device and generate electrical voltage signals that are recorded and used for machine learning classification.

---

## Experimental Setup

The experimental system was constructed as follows:

1. A **speaker** was used to generate sound waves corresponding to specific command words.
2. A **hollow cylindrical base** was mounted on top of the speaker.
3. A **TENG device** was placed on top of this cylindrical structure.
4. Sound waves from the speaker produce **mechanical vibrations**.
5. These vibrations excite the **TENG sensor**, generating electrical voltage signals.
6. The generated signals were recorded and stored as **CSV files**.

System pipeline:

Speaker (sound waves)  
↓  
Mechanical Vibrations  
↓  
TENG Device  
↓  
Electrical Voltage Signal  
↓  
CSV Signal Dataset  
↓  
1D CNN Classification

## Command Words Used

The dataset contains voltage signals corresponding to the following **10 command words**:

ASCEND  
DIRECTION  
HELP  
LOW AIR  
NO WORD  
OK  
OUT OF AIR  
PROBLEM  
SAVE  
STOP  

Each command produces a **distinct vibration pattern**, which results in a **unique electrical signal from the TENG sensor**.

## Signal Processing

Before training the model:

• Signals are normalized  
• Signals are trimmed or zero padded to a fixed length  

TARGET_LENGTH = 16000

This ensures that all signals have a **consistent input size for the neural network**.

---

## Neural Network Architecture

A **1D Convolutional Neural Network** was used to classify the vibration signals.

Architecture summary:

Input: Voltage Signal (16000 samples)

Conv1D (64 filters)  
BatchNorm  
ReLU  
MaxPool  

Conv1D (128 filters)  
BatchNorm  
ReLU  
MaxPool  

Conv1D (256 filters)  
BatchNorm  
ReLU  
MaxPool  

Conv1D (512 filters)  
BatchNorm  
ReLU  
MaxPool  

Adaptive Average Pooling  

Fully Connected Layers  
512 → 256 → 10 output classes  

The network learns **temporal patterns in vibration signals** produced by different spoken commands.

---

## Training Configuration

The model was trained using:

Loss Function: CrossEntropyLoss  
Optimizer: AdamW  
Learning Rate: 0.0003  
Batch Size: 16  
Epochs: 70  

Test-Time Augmentation (TTA) was used during validation and testing to improve prediction stability.

---

## Results

The trained model achieved the following performance:

Validation Accuracy: **99.09%**  
Test Accuracy: **99.09%**

The confusion matrix confirms that the CNN successfully distinguishes between vibration signatures generated by different command words.
